# LaflaGPT 380M Lightning H200 Training Plan

## Decision

Use H200 first when the repo archive and launch script are ready before the
machine starts. A100 is the fallback only if H200 setup/availability burns more
than about 20 minutes. H100 for one hour is too short for repeated setup,
checkpoint testing, and recovery. L40S is acceptable only as a long fallback.
T4 is not appropriate for this run.

## Base Training

- Model: `configs/model/lafla-380m-thinking.yaml`
- Training: `configs/training/lightning-h200-380m-50000.yaml`
- Runtime target: `configs/runtime/desktop-phone-fp16-380m.yaml`
- Identity seed: `configs/data/lafla-model-identity-380m.jsonl`
- Real-data target for the first H200 run: 2.2B characters, minimum 900M.
- Checkpoints: every 250 steps, keep last 4.

The 50000-step number is the long target across resumable sessions, not a claim
that one free H200 window will finish it. A single H200 window should prioritize
getting a clean base checkpoint, loss curve, tokenizer report, and sample tests.

## Data Mix

The first pass is Turkish-heavy and adds enough English, math, and code to avoid
turning the model into a narrow memorizer:

- FineWeb2 Turkish and FineWeb2-HQ Turkish for broad Turkish language coverage.
- Wikimedia Turkish for factual, encyclopedia-style Turkish.
- FineWeb2 English for bilingual transfer.
- OpenWebMath for math language and reasoning surface area.
- The Stack smol Python subset for code syntax and documentation style.
- Optional local reviewed defensive-security docs.
- Aya Turkish instruction and Lafla-owned identity records.

No synthetic data is generated by the preparation script. Optional local sources
are skipped and reported if missing.

## Thinking Timing

Do not mix raw chain-of-thought style data into early base pretraining. First
train the base model to produce stable Turkish, English, code, and math text.
Then add a small supervised thinking/instruction phase with private thinking
tags and public-answer cleanup. Thinking SFT should start after a checkpoint
that already passes basic completion tests; otherwise it teaches the model to
format bad reasoning instead of improving reasoning.
